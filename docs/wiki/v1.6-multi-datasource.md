# v1.6 多数据源 + 连接密文管理（1.5 收官后置）

> 背景：1.5 可演示；后续分库分表时，**每个页面/查询应能绑定数据源**，且 **连接串与密码不得明文落库**。  
> 本版：**元模型 + 加密保险箱 + 管理 API** 先落地；**运行时按 ds 路由** 下一切片接 Query/Action/Page CRUD。

原则仍成立：

- **元数据库**（`lc_*`）与 **业务库** 可分离  
- 业务 SQL 文本保持领域纯净，**不写库名拼接到每条 SQL**（用运行时绑定）  
- 权限：`perm:config` 才能管连接；密码 **只写不读**

---

## 1. 目标

| 能力 | 说明 |
|------|------|
| 连接目录 | 多个 JDBC 数据源配置（PG 优先，可扩 MySQL 等） |
| 密文存储 | 密码 AES-GCM 加密；主密钥来自环境变量，不进 Git |
| 页面/查询绑定 | `page` / `query` 可选 `data_source_code` |
| 解析顺序 | **Page 覆盖 > Query 绑定 > 平台默认 `default`** |
| 平台库 | `is_platform=true` 的 `default` 始终走 Spring Boot 主数据源（当前演示库） |

---

## 2. 表模型

### 2.1 `lc_data_source`

```text
ds_code            PK          -- default | biz_shop | report_ro ...
name               显示名
driver_class       org.postgresql.Driver | ...
jdbc_url           jdbc:postgresql://...
username
password_cipher    可空；AES-GCM(Base64: iv||ciphertext||tag)
max_pool_size      默认 5
enabled
is_platform        true 时运行时忽略 cipher，用 spring.datasource
remark
created_at / updated_at
```

### 2.2 绑定列

```text
lc_page_model.data_source_code   → lc_data_source(ds_code) 可空
lc_query_model.data_source_code  → lc_data_source(ds_code) 可空
```

**解析：**

```text
resolveDs(page, query) =
  page.data_source_code
  ?? query.data_source_code
  ?? "default"
```

分库分表时：同一业务域可共用一个 `ds_code`；分表策略仍在 SQL/中间件层，**不进密码箱**。

---

## 3. 加密方案

| 项 | 选择 |
|----|------|
| 算法 | **AES-256-GCM** |
| 主密钥 | 环境变量 / `lowcode.datasource.crypto-key`（Base64 或 64 hex） |
| 密文格式 | Base64( IV 12B ‖ ciphertext+tag ) |
| 轮换 | 改主密钥后需批量重加密（管理台「轮换」后续做） |
| 禁止 | 日志打印明文密码；GET API 返回 password |

开发机若未配置密钥：使用 **仅本地** 的派生密钥并打 WARN（不可用于生产）。

生产：

```bash
export LOWCODE_DS_CRYPTO_KEY="$(openssl rand -base64 32)"
```

## 4. 运行时路由（下一切片）

接入点（现有单 `NamedParameterJdbcTemplate` 改为路由器）：

1. `QueryEngineService.execute*`  
2. `ActionService` sqlTransaction  
3. `PageService` 单表 CRUD  

```text
DataSourceRegistry.get(dsCode) → NamedParameterJdbcTemplate
  - default / is_platform → primary bean
  - 其它 → 懒创建 Hikari 池（缓存），密码 decrypt 仅内存
```

**元数据读写**（`lc_*`）永远走平台库，避免鸡生蛋。

---

## 5. 管理 API（本版）

| 方法 | 路径 | 权限 |
|------|------|------|
| GET | `/api/v1/admin/data-sources` | perm:config |
| GET | `/api/v1/admin/data-sources/{code}` | 同上（无密码） |
| POST | `/api/v1/admin/data-sources` | 同上 |
| PUT | `/api/v1/admin/data-sources/{code}` | 同上；password 可选更新 |
| DELETE | `/api/v1/admin/data-sources/{code}` | 禁止删 `default` |
| POST | `/api/v1/admin/data-sources/{code}/test` | 试连（临时连接，不落池） |

前端入口：系统 **数据源**（与 SQL Repo / 权限并列，仅 owner）。

---

## 6. 与 1.5 关系

- **现在**：演示与业务仍全部在平台库；绑定列可空 = 默认  
- **配置**：可先登记未来业务库连接（密文），不强制切换  
- **切换执行**：实现 Registry 后，改 page/query 的 `data_source_code` 即可，无需改 SQL 文本

## 8. 不做

- 应用层分表中间件（ShardingSphere 等）——可选插件，不绑死  
- 把密码写进 `application.properties` 多套 spring.datasource.* 做唯一方案（难动态增删）  
- 在 SQL 资产里硬编码 `dbname.table` 代替数据源绑定
