# 后续开发 Wiki

## 后续重点

### 1. 把 SQL 作为唯一第一入口

任何新页面都应该优先满足：

- 先写 SQL
- 立刻有表格
- 再补页面配置
- 最后才补实体语义和写能力

### 2. 收敛 EntityModel

`EntityModel` 要从“默认字段列表”演化为“实体语义模型”。

### 3. CRUD 分层

- 查询：默认开放
- 写入：按实体语义校验后开放
- 复杂写入：交给自定义 action 或后续 DSL

### 4. 编辑器能力

后续建议引入 Monaco：

- SQL 高亮
- JSON schema 校验
- 错误提示
- 自动格式化

### 5. 页面形态

配置态与运行态继续分化：

- 运行态更像产品
- 配置态更像 studio

## 维护约定

每次做下面几类改动时，都应该同步更新 wiki：

- 模型结构变化
- CRUD 规则变化
- DSL 变化
- 配置态布局变化
- 部署方式变化
