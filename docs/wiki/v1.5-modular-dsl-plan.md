# v1.5 模块化 DSL 方案

## 版本定位

v1.5 的目标是把当前阶段从“能展示的 SQL 表格 demo”推进到“可继续扩展的 SQL-driven SaaS 前端框架雏形”。

这一版不追求复杂主子表、流程管理或任意脚本平台，而是先把底座分清楚：

- 通用 DSL 能力独立于具体模板
- 数据源、控件、过滤器、动作、插件都是公共模块
- 当前单表页面只是众多模板中的第一个模板
- SQL 驱动仍然是第一入口
- CRUD 只在主键和单实体语义明确时启用

## 核心原则

### 1. Template 不是 DSL 本体

模板只是把用户配置编译成标准运行时结构的一种方式。

当前需要支持的模板：

- `rawSqlTemplate`
- `singleTableTemplate`

后续可以继续增加：

- `joinReadonlyTemplate`
- `masterDetailTemplate`
- `dashboardLiteTemplate`
- `workflowTemplate`

运行时不应该被某个模板绑死。新增模板时，尽量只新增模板编译器，而不是重写表格、表单、过滤器和动作运行时。

### 2. 控件和数据源是 Core DSL

以下能力必须独立于模板：

- `DataSourceModel`
- `QueryModel`
- `EntityModel`
- `PageModel`
- `FieldModel`
- `EditorModel`
- `FilterModel`
- `ActionModel`
- `PluginModel`

例如 `select`、`autocomplete`、`date`、`datetime` 控件不能只服务于 `singleTableTemplate`。后续主子表、join 只读表、dashboard 容器页、流程页面也应该复用同一套控件定义。

### 2.1 Dashboard Lite 也是模板，不是页面设计器

Dashboard 类页面只增加一种模板编译器：把固定容器 DSL 编译成运行时组件树。

它应复用：

- `QueryModel`
- `ActionModel`
- `PluginModel`
- `I18n`
- `CacheProvider`
- `AuthzGateway`

它不应引入：

- 拖拽画布
- 任意 CSS 编辑
- 专用 dashboard 数据 API
- 绕过 SQL 仓库的组件数据源

### 3. SQL-first，但分模式

SQL 仍然是第一入口，但不同模式承诺不同能力：

- `rawSql`：自由 SQL，默认只读，适合查询、调试、远端数据操作入口
- `singleTableTemplate`：结构化单表模板，支持筛选、排序、分页和受控 CRUD

不要把任意 SQL 结果集承诺成可写页面。

### 4. CRUD 必须依赖主键

在 SQL 驱动场景中，编辑和删除必须依赖稳定主键。

CRUD 启用条件：

- 绑定单一 `anchorEntity`
- 能确认 `primaryKey`
- 查询结果包含主键字段
- 写回目标表明确
- 查询不是聚合、汇总、union 或不可逆结果集

不满足条件时，页面应降级为只读 smart grid。

### 5. JS 扩展是前端插件，不是流程引擎

v1.5 可以让 DSL 声明 JS 资源路径，用于控制控件外观、字段行为和自定义 action。

但流程状态控制、后端规则执行、复杂生命周期钩子先不放进 v1.5。它们应该留给后续 workflow/Groovy 版本。

## 分层架构

### Core DSL

负责描述稳定公共能力：

- 数据源
- 查询引用
- 字段
- 控件
- 过滤器
- action
- 插件资源
- 实体语义

### Template Layer

负责把不同模板配置编译成标准运行时结构：

- `rawSqlTemplateCompiler`
- `singleTableTemplateCompiler`
- 后续 `masterDetailTemplateCompiler`
- 后续 `joinReadonlyTemplateCompiler`

模板层可以有特化规则，但输出应尽量靠近统一的 runtime contract。

### Runtime Engine

负责执行标准 DSL：

- `gridEngine`
- `formEngine`
- `filterEngine`
- `actionEngine`
- `editorRegistry`
- `optionProvider`
- `pluginLoader`

运行时只关心标准结构，不应该散落大量 `if templateType == ...` 的逻辑。

### Extension Layer

负责加载和注册扩展：

- 前端 JS plugin
- 自定义 editor renderer
- 自定义 action handler
- 后续 Groovy action / workflow hook

## v1.5 模型草案

### DataSourceModel

数据源独立配置，供列表查询、下拉选项、自动补全、预加载和未来缓存复用。

```json
{
  "code": "main_pg",
  "type": "postgres",
  "name": "Main PostgreSQL",
  "enabled": true,
  "config": {
    "schema": "public"
  },
  "cache": {
    "enabled": false,
    "ttlSeconds": 60
  }
}
```

第一版可以继续使用后端默认数据库连接，但 DSL 结构上先预留 `dataSource`。

### QueryModel

```json
{
  "queryCode": "customer_list",
  "mode": "singleTableTemplate",
  "dataSource": "main_pg",
  "template": {
    "tableName": "customer",
    "primaryKey": "id",
    "select": ["id", "code", "name", "status", "created_at"],
    "baseWhere": "status <> -1",
    "defaultOrderBy": "id desc"
  }
}
```

自由 SQL 模式：

```json
{
  "queryCode": "customer_debug",
  "mode": "rawSql",
  "dataSource": "main_pg",
  "sqlText": "select id, code, name from customer where status = 1"
}
```

后续可增加 `sqlFile`：

```json
{
  "queryCode": "customer_report",
  "mode": "rawSql",
  "dataSource": "main_pg",
  "sqlFile": "/queries/customer_report.sql"
}
```

### PageModel

```json
{
  "pageCode": "customer_page",
  "title": "Customers",
  "queryCode": "customer_list",
  "writable": "auto",
  "columns": [
    { "field": "code", "title": "Code", "width": 140 },
    { "field": "name", "title": "Name", "editor": "customerName" },
    { "field": "status", "title": "Status", "editor": "statusSelect" },
    { "field": "created_at", "title": "Created", "format": "datetime" }
  ],
  "filters": [
    { "field": "name", "operator": "ilike", "editor": "customerName" },
    { "field": "status", "operator": "=", "editor": "statusSelect" },
    { "field": "created_at", "operator": "between", "editor": "createdRange" }
  ],
  "actions": [
    { "code": "refresh", "type": "builtin" },
    { "code": "edit", "type": "builtin", "requiresWritable": true },
    { "code": "delete", "type": "builtin", "requiresWritable": true }
  ],
  "plugins": [
    { "code": "customer-ui", "script": "/extensions/customer-ui.js" }
  ]
}
```

### EditorModel

控件定义独立出来，供不同模板复用。

```json
{
  "editors": {
    "customerName": {
      "type": "text",
      "placeholder": "Customer name",
      "suggest": {
        "enabled": true,
        "source": "sql",
        "queryCode": "customer_name_suggest",
        "keywordParam": "keyword",
        "minChars": 2
      }
    },
    "statusSelect": {
      "type": "select",
      "options": {
        "source": "static",
        "items": [
          { "label": "Active", "value": 1 },
          { "label": "Disabled", "value": 0 }
        ]
      }
    },
    "createdRange": {
      "type": "datetimeRange",
      "locale": "page",
      "timezone": "browser"
    }
  }
}
```

下拉数据源可以来自 SQL：

```json
{
  "type": "select",
  "options": {
    "source": "sql",
    "queryCode": "country_options",
    "labelField": "name",
    "valueField": "code",
    "preload": true,
    "cacheTtlSeconds": 300
  }
}
```

后续可以扩展：

```json
{
  "type": "select",
  "options": {
    "source": "decorator",
    "decorator": "countryOptionsProvider"
  }
}
```

### EntityModel

实体模型不再承担页面列定义职责，而是描述实体语义。

```json
{
  "code": "customer",
  "tableName": "customer",
  "primaryKey": "id",
  "labelField": "name",
  "fields": [
    { "field": "id", "readonly": true, "hidden": true },
    { "field": "code", "required": true },
    { "field": "name", "required": true },
    { "field": "status", "dictionary": "status" },
    { "field": "created_at", "readonly": true, "audit": true }
  ]
}
```

## 过滤器执行策略

### singleTableTemplate

单表模板由后端生成稳定 SQL：

```sql
select <columns>
from <table>
where 1 = 1
  and (<baseWhere>)
  and <filter conditions>
order by <sort>
limit <limit>
offset <offset>
```

这样不依赖用户在原始 SQL 后手写 `and 1=1`。

### rawSql

自由 SQL 默认只读。筛选可以先通过外层包裹实现：

```sql
select *
from (
  <raw sql>
) t
where 1 = 1
  and <filter conditions>
```

rawSql 的筛选只做读取增强，不开启通用写入。

## 前端 JS 插件边界

v1.5 支持：

- 通过 DSL 声明 JS 路径
- 注册自定义 editor
- 注册自定义 action
- 注册字段级行为

v1.5 不支持：

- 任意覆盖运行时核心
- 任意后端写入
- 流程状态机
- Groovy 流程控制

建议插件 API 保持窄接口：

```ts
export default function registerEasySaasPlugin(api) {
  api.registerAction("openCustomerPortal", async (ctx) => {
    window.open(`/customers/${ctx.row.id}`, "_blank");
  });
}
```

## v1.5 实施顺序

1. 提取通用 DSL 类型和 schema
2. 引入 `mode = rawSql | singleTableTemplate`
3. 增加 `dataSource` 引用字段，先绑定默认数据库
4. 实现 `singleTableTemplate` 查询生成
5. 修复过滤器，统一支持分页、排序和筛选
6. 实现主键校验后的 edit/delete
7. 引入 editor registry
8. 支持 `text/select/autocomplete/date/datetime/number`
9. 支持 static/sql options 和 suggest
10. 支持前端 JS plugin 路径加载

## v1.5 不做

- join 查询可写
- 完整主子表详情结构
- 多层嵌套 tab
- Groovy 流程控制
- 通用流程引擎
- 任意 SQL 自动 CRUD

## 验收口径

v1.5 完成时，应满足：

- 只写 SQL 仍可展示只读表格
- 单表模板可稳定支持筛选、排序、分页
- 单表模板在主键明确时可编辑和删除
- 控件定义可在不同页面和模板复用
- 下拉和自动补全支持静态数据与 SQL 数据源
- DSL 已经具备后续主子表、join 只读、流程页扩展空间
