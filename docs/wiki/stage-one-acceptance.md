# 阶段一验收说明

## 目标

阶段一不是做完整低代码平台，而是交付一个可展示的、`SQL-first` 的智能表格工作台。

核心链路：

`SQL / QueryModel -> Smart Grid Preview -> PageModel 增强 -> EntityModel 语义增强`

## 阶段一必须成立的体验

### 1. 只配置 SQL 就能先出表格

这是阶段一最核心的验收点。

最小路径应该成立：

1. 配置一个 SQL
2. 系统自动识别结果列
3. 自动显示分页表格
4. 自动支持基础排序
5. 自动生成默认页面配置草案

如果这条链路不成立，产品就还没有真正达到“web 版 SQL 客户端 + 智能表格模板”的目标。

### 2. PageModel 是增强层

`PageModel` 用来覆盖：

- 列顺序
- 页面显示名
- 过滤器
- 动作
- 空态文案
- 分页/密度/展示布局

它不应该是第一步必填配置。

### 3. EntityModel 是语义增强层

`EntityModel` 用来补充：

- 主键
- 实体说明
- 默认 label 字段
- 默认 id 字段
- 只读/隐藏/审计/字典等语义
- CRUD 需要的稳定实体信息

它不应该再承担一份和 `PageModel.columns` 重复的页面显示职责。

### 4. Runtime 与 Config 必须分离

- `Runtime` 负责展示业务页面
- `Config` 负责搭建、编辑和调试

配置态不应该让大表格压在最上面，表格应该是预览，而不是配置主视图。

## 阶段一建议验收清单

### P0

- 只配 SQL 能直接生成只读表格
- 服务端分页正常
- 排序正常
- 筛选正常
- Raw SQL 入口保留
- 配置态和运行态分离
- 页面可作为展示站点使用

### P1

- 保存 SQL 后自动生成默认 `PageModel`
- 保存 SQL 后自动推导默认 `EntityModel` 草案
- JSON / SQL 基础语法检查
- 预览面板可折叠

### P2

- 更强的编辑器能力
- 更稳定的实体语义配置
- 更明确的 CRUD 启用条件

## 当前阶段结论

阶段一应当把“只配 SQL 就出表格”视为主卖点。

而 CRUD 只应该是第二层能力，不应反过来定义第一阶段产品。
